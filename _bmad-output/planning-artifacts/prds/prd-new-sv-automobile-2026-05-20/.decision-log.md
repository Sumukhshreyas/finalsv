# Decision Log — SV Enterprises Product Showcase Website

## 2026-05-20 — PRD Creation

### D-1: No E-Commerce

**Decision:** The website will have zero e-commerce functionality — no cart, checkout, payment, or order tracking.
**Rationale:** The business model is offline wholesale/dealer. The website is a product showcase and enquiry generator. All transactions happen via WhatsApp, phone, or in-person.
**Status:** Final.

### D-2: Reference HTML as Fixed Design Spec

**Decision:** The HTML file at `reference/index.html` is treated as the sacred, immutable UI/UX specification. Development must replicate it exactly.
**Rationale:** The design is already complete and approved. Changing it would delay the project and risk losing the carefully crafted responsive and visual design.
**Status:** Final.

### D-3: Mobile-First Development

**Decision:** All development follows a mobile-first approach, with the primary target device being smartphones at 360–430px viewport width.
**Rationale:** The majority of users (workshop mechanics, industrial buyers) will access the website on mobile devices.
**Status:** Final.

### D-4: SEO-First Architecture

**Decision:** The website must use SSR or SSG to ensure search engines receive fully rendered HTML. No hash routing. Clean URLs required.
**Rationale:** The primary business goal is organic Google ranking and Google Ads landing page support. Client-side-only rendering would hurt SEO.
**Status:** Final.

### D-5: URL Structure

**Decision:** Category pages use `/{mode}/{category-slug}/` and product pages use `/{mode}/{category-slug}/{product-slug}/`. Custom slug overrides are supported.
**Rationale:** Short, readable, keyword-rich URLs improve SEO and user experience. The two-mode (automobile/industrial) prefix ensures clean content separation.
**Status:** Final.

### D-6: Static Product Data for v1

**Decision:** Product data will be static (JSON/JS objects as in the reference HTML) for v1. No CMS or database backend.
**Rationale:** Simplifies initial deployment. A CMS can be added in v2 when the product catalog grows and requires frequent updates.
**Status:** Final — revisit for v2.

### D-7: PRD Fast Path

**Decision:** PRD created via fast path using the user's detailed requirements and reverse-engineering of the reference HTML.
**Rationale:** User provided comprehensive requirements including business goals, URL structure, product display fields, and guardrails. The reference HTML provided the remaining feature specification.
**Status:** Final.
