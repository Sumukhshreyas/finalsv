## Dev Notes

### Current Workspace State

- The repository root currently contains planning and reference folders only; no `package.json` or `src/` app exists yet.
- This story creates the application foundation. It should not implement catalogue pages, data files, UI extraction, routing, or components beyond the starter shell unless the starter generates defaults.
- Do not delete or overwrite BMad artifacts or `reference/index.html`.

### Architecture Guardrails

- Data is JSON-backed, not Firebase-backed. Do not install or configure Firebase, Firestore, Firebase Storage, Prisma, or any database SDK.
- Future catalogue data will live in `src/data/catalog.json` and be queried through `src/lib/dataUtils.ts`. That is Story 1.3/1.4 work, not this story.
- The site is a public enquiry-generation catalogue. Do not add auth, login, cart, checkout, payments, order tracking, or API endpoints.
- Use TypeScript and Next.js App Router with static/SSR-friendly defaults.
- Preserve the reference design constraint for later stories: `reference/index.html` is the approved visual source of truth.

### Required Starter Command

```bash
npx create-next-app@latest ./ --typescript --eslint --app --no-tailwind --src-dir --turbopack
```

If the CLI prompts for options despite flags, choose App Router, TypeScript, ESLint, `src/` directory, no Tailwind, and Turbopack where offered.

### Security Headers

Add headers through Next.js config using the `headers()` function. Required headers:

- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Content-Security-Policy`: start conservative but compatible with local Next.js development and later Google Fonts/Vercel Analytics needs. If the strict policy breaks the starter app, document the reason and adjust narrowly.

### Environment File

Create `.env.example` for optional settings only. Suggested initial content:

```text
NEXT_PUBLIC_SITE_URL=https://example.com
```

Do not include Firebase keys, database URLs, service account values, or storage bucket values.

### Sitemap Setup

Create `next-sitemap.config.js`. The production domain is not finalized, so use `process.env.NEXT_PUBLIC_SITE_URL || 'https://example.com'` for `siteUrl`.

Do not hand-write `public/sitemap.xml` or `public/robots.txt`; those are generated by `next-sitemap`.

### Testing / Verification

Run these commands before moving the story to review:

```bash
npm run dev
npm run build
```

If dependency installation requires network access or approval, request it rather than replacing packages with unrelated alternatives.
